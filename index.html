What are JS Functions?

Functions are a fundamental aspect of JavaScript, a programming language. In JavaScript, functions are structured blocks of code specifically designed to carry out a particular task or set of tasks. When a function is called upon or 'invoked', it then executes the task it was designed for.

The way to define a function in JavaScript is by using the 'function' keyword. This is then followed by a unique name given to the function. The name is followed by a set of parentheses (). The rules for naming functions are similar to the rules for naming variables in that function names can contain letters, digits, underscores, and dollar signs.

Inside the parentheses, parameter names can be included and these are separated by commas for clarity (parameter1, parameter2, etc.). Following the parentheses, the specific code that the function is designed to execute is placed inside curly brackets: {}.

For instance, let's consider a simple JavaScript function whose task is to add two numbers together. It could be defined like this: function addNumbers(a, b) { return a + b; }. To invoke this function, you would call it with two numbers as arguments, like so: addNumbers(3, 4). The function would then perform its task and return the result of adding the two numbers together.

In JavaScript, functions can also be assigned to variables. This is known as a function expression. A function expression does not necessarily need to be given a name, it can be defined as an anonymous function. For instance, let x = function (a, b) {return a * b}; is an anonymous function that multiplies two numbers. To call this function, you would use the variable name, like so: x(4, 5). This will return the result of 4 multiplied by 5.

Furthermore, JavaScript introduces an interesting concept known as the callback function. In simple terms, a callback function is a function that is passed into another function as an argument. The callback function is then invoked inside the outer function. This concept is a key component of asynchronous programming in JavaScript as it allows the execution of code to continue without having to wait for the function that was passed as a callback to complete.

Last but not least, JavaScript also supports a more recent feature known as arrow functions, introduced in ES6. Arrow functions offer a more concise syntax to write function expressions. They are particularly useful for non-method functions and it's important to note that they cannot be used as constructors.

Types of functions

Named Functions: These are the standard function that we define using the functions keyword followed by a name. The name is followed by parentheses, which can include parameters separated by commas. The code to be executed by the function is placed inside curly brackets {}. An example of a named function is: function addNumbers(a, b) { return a + b; }. This function is called with two numbers as arguments and it returns the sum of these two numbers.

Function Expressions: These are functions that are assigned to variables. The function itself can be anonymous, which means it does not necessarily need a name. The function is defined and then assigned to a variable. When calling this type of function, the variable's name is used. For instance, let x = function (a, b) { return a * b }; is an example of an anonymous function expression that multiplies two numbers. To use this function, you would call it with the variable name and two numbers as arguments like this: x(4, 5). This will return the result of 4 multiplied by 5.

Callback Functions: These are functions that are passed into another function as an argument and then invoked inside the outer function. This is a fundamental aspect of asynchronous programming in JavaScript, which allows the execution of the code to continue without waiting for the function that was passed as a callback to complete. Callback functions can help to manage and control the flow of the program, especially when dealing with asynchronous operations.

Arrow Functions: Arrow functions were introduced in ES6 as a new way to write shorter function syntax. They are particularly useful when a function is placed into a variable - making it a function expression. Arrow functions are anonymous and change the way this binds in functions. They are best suited for non-method functions, and they cannot be used as constructors. An example of an arrow function is: const x = (a, b) => a * b;. This function multiplies two numbers and returns the result.

Scope and Stack

In JavaScript, each function creates a new scope. Scope determines the visibility or accessibility of variables, functions, and objects in some particular part of your code during runtime. In other words, it defines the portion of the code where a variable or a function can be accessed.

Let's consider a simple example:

                            
                                function exampleFunction() {
                                    let x = 10;
                                    if (true) {
                                      let y = 20;
                                      console.log(x + y);  // This will output 30
                                    }
                                    console.log(y);  // This will output 'undefined'
                                  }

In this example, x is accessible within the entire function exampleFunction(), but y is only accessible within the if block.

The JavaScript engine executes the function code from top to bottom, and as it executes, it creates a execution context. For each function call, a new execution context is stacked on top of the previous one, creating a call stack. When a function finishes execution, its execution context is popped off from the stack, and control reaches to the context below in the current stack.

Consider the following example:

                            
                                function firstFunction() {
                                    console.log('First');
                                    secondFunction();
                                    console.log('Back in first function');
                                  }
                                  
                                  function secondFunction() {
                                    console.log('Second');
                                  }
                                  
                                  firstFunction();

In this example, calling firstFunction() creates a first execution context which is put on the call stack. When secondFunction() is called inside firstFunction(), a second context is created and put on top of the first one in the call stack. When secondFunction() finishes executing, its context is popped from the stack, and execution continues in the firstFunction() context.

Nested Functions

Nested functions in JavaScript are functions that are defined inside other functions. They are also known as inner functions. The outer function that encloses the inner function is called the parent function. The inner function can only be accessed from statements in the outer function.

Nested functions are an important feature of JavaScript because they allow for information hiding and encapsulation, which can lead to better-organized and more maintainable code. Inner functions have access to the variables and parameters of the outer (parent) function. This is known as lexical scoping.

Here is a simple example of a nested function in JavaScript: 

                            
                                function outerFunction(a, b) {
                                    function innerFunction(c) {
                                      return a * b * c;
                                    }
                                    return innerFunction;
                                  }
                                  
                                  let result = outerFunction(2, 3)(4);

In this example, innerFunction is nested within outerFunction. innerFunction is able to access the parameters a and b from its parent function, outerFunction. The outerFunction is then called with two arguments, and it returns the innerFunction. The returned innerFunction is immediately called with the argument 4, and it returns the result of the multiplication of 2, 3, and 4.

Remember that functions in JavaScript are first-class objects. This means that they can be passed as arguments to other functions, returned as values from other functions, assigned to variables, or stored in data structures.

Closures

n JavaScript, a closure is a function that has access to its own scope, the outer function’s scope, and the global scope, as well as access to its own arguments, the outer function’s arguments, and the global arguments. A closure gives you access to an outer function’s scope from an inner function.

In JavaScript, closures are created every time a function is created, at function creation time. To use a closure, define a function inside another function and expose it. To expose a function, return it or pass it to another function.

Here is a simple example:

                            
                                function outerFunction(outerVariable) {
                                    return function innerFunction(innerVariable) {
                                      console.log('outerVariable:', outerVariable);
                                      console.log('innerVariable:', innerVariable);
                                    }
                                  }
                                  
                                  const newFunction = outerFunction('outside');
                                  newFunction('inside');

In this example, outerFunction is creating and returning the innerFunction. The innerFunction is a closure that is closed over outerVariable. This closure is a function (innerFunction) bundled together with its lexical environment. The lexical environment consists of any local variables that were in-scope at the time the closure was created. In this case, outerVariable is part of the newFunction closure.
